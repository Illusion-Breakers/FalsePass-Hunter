## 项目概述

### 核心价值主张

**FalsePass Hunter** 是一个面向智能制造领域的质量分析平台，基于 **Kaggle UCI SECOM** 真实数据集构建。

**解决的核心问题：** 在产品出厂前，把那些"测试装正常但实际有毛病"的次品挑出来。

### 什么是 False Pass（虚假通过）？

```
传统测试逻辑：
- 参数在规格范围内 → Pass ✅
- 参数超出规格范围 → Fail ❌

问题：
- 某产品参数没超限，但已经偏离"健康状态"
- 传统测试：Pass ✅
- 实际：这是 False Pass，有隐患 ⚠️
```

### 检测原理（核心算法）

```
第 1 步：分组
- Pass 组：1500 个正常产品
- Fail 组：67 个问题产品

第 2 步：特征筛选
- 计算每个参数的 Cohen's d（效应量）
- 筛选出区分度高的敏感参数

第 3 步：建立基线
- 用 Pass 组计算均值和标准差
- 健康范围 = 均值 ± 3×标准差

第 4 步：异常检测
- 新产品参数 vs 健康基线
- Z-score > 3 → 预警 ⚠️
```

## 常见问题 Q&A

### Q1: 你们的数据是真实的吗？

> "我们使用 Kaggle UCI SECOM 数据集，这是半导体制造的真实传感器数据。演示界面为了清晰展示逻辑，部分数据是模拟的。接入真实产线后，数据会从后端 API 实时获取。"

### Q2: 你们怎么检测 False Pass？

> "核心逻辑是和'正常产品'对比。我们分析 SECOM 数据集中 Pass 产品的参数，建立健康基线（均值±3 倍标准差）。新产品来了，计算每个参数的 Z-score，超过 3 个标准差就预警。
>
> 举个例子：某电压参数，正常产品均值 3.30V，标准差 0.03V，健康范围是 3.21V~3.39V。新产品测到 3.55V，虽然没超规格限（3.6V），但偏离健康范围，我们预警。"

### Q3: AI 体现在哪里？

> "AI 不是噱头。我们用特征工程从 592 个参数里提取敏感特征，用降维算法减少计算量，最后用异常检测模型识别 False Pass 模式。日志分析模块用 LLM 理解工程文本。"

### Q4: 你们项目解决了什么问题？

> "传统测试是'及格线思维'（60 分万岁），我们是'健康度思维'（虽然及格但已经偏离正常状态）。我们解决的是灰色地带的问题——那些测试通过了，但参数已经在临界值附近的产品。"
